## 1. О проекте

Cox Probability System (CPS) — пользовательский индикатор для MetaTrader 5, реализующий байесовскую вероятностную систему с фильтром энтропии Шеннона. Индикатор **не даёт прямых сигналов на покупку/продажу**, а предоставляет трейдеру визуальную информацию о текущем состоянии рынка через:

- **Вероятностную линию** (Signal Probability) — апостериорную байесовскую вероятность движения цены **вверх** (значение от 0 до 1)
- **Линию энтропии** (Entropy) — меру неопределённости рынка по Шеннону
- **Цветовые зоны** — серый фон в периоды высокой неопределённости, когда принимать решения **не рекомендуется**
- **Текстовую метку** — оперативную сводку текущего состояния

### 1.1. Математическая основа

Индикатор базируется на трёх ключевых концепциях:

| Концепция | Описание |
|---|---|
| **Байесовское обновление** | Априорные вероятности (посчитанные по историческим бычьим/медвежьим барам) корректируются функцией правдоподобия (насколько часто одно направление следует за таким же). Результат — апостериорная вероятность. |
| **Энтропия Шеннона** | `H = −p·log₂(p) − (1−p)·log₂(1−p)`. Максимум (1.0) при p = 0.5 — полная неопределённость. Минимум (0.0) при p = 0 или p = 1 — полная уверенность. |
| **Энтропийный фильтр** | Сигнал считается **действительным** только когда энтропия ниже заданного порога (по умолчанию 0.65). Высокая энтропия = «рынок шумит» = лучше не торговать. |

## 2. Как установить

1. Скачайте файл `Cox_Probability_System.mq5` из этого репозитория.
2. Скопируйте его в папку индикаторов MetaTrader 5:
   - **Windows:** `C:\Program Files\MetaTrader 5\MQL5\Indicators\`
   - **macOS/Linux (через Wine/CrossOver):** аналогичный путь внутри бутылки/префикса
3. В MetaTrader 5 откройте **MetaEditor** (F4), найдите файл во вкладке «Навигатор» → «Indicators», нажмите **«Compile»** (F7).
4. После успешной компиляции перезапустите терминал или нажмите «Refresh» в окне навигатора.
5. Индикатор появится в списке пользовательских индикаторов (вкладка «Indicators» в окне навигатора). Перетащите его на график.

## 3. Как пользоваться

### 3.1. Что показывают линии

| Линия | Цвет | Значение |
|---|---|---|
| **Signal Probability** | Синяя (DodgerBlue) | Сглаженная вероятность движения **вверх** на следующем баре. Значения выше 0.65 — смещение вверх, ниже 0.35 — вниз, около 0.50 — нейтралитет. |
| **Entropy** | Оранжево-красная (OrangeRed) | Сглаженная энтропия Шеннона. Высокие значения (> 0.65) = рынок хаотичен, сигнал ненадёжен. |

### 3.2. Горизонтальные уровни

| Уровень | Цвет/Стиль | Значение |
|---|---|---|
| `MinProbabilityToTrade` (по умолч. 0.65) | Зелёный пунктир | Минимальная вероятность для входа. Синяя линия выше этого уровня + энтропия низкая → вероятность вверх высока. |
| `0.50` | Серый тонкий | Нейтралитет. |
| `EntropyThreshold` (по умолч. 0.65) | Красный пунктир | Порог энтропии. Оранжевая линия ниже этого уровня → рынок определён, можно доверять сигналу. |

### 3.3. Фоновая закраска

Если мгновенная (не сглаженная) энтропия **превышает** `EntropyThreshold`, фон за текущим баром закрашивается **серым цветом** (`HighEntropyBgColor`). Это визуальный фильтр: **в серой зоне сигналы ненадёжны, лучше воздержаться от сделок**.

При `ShowEntropyZone = false` закраска отключается.

### 3.4. Текстовая метка (левый верхний угол)

В реальном времени отображает:

- **Entropy RAW:** мгновенная (несглаженная) энтропия текущего бара
- **Signal:** мгновенная апостериорная вероятность
- **Trade allowed:** `YES` — если энтропия ниже порога И вероятность достигла минимума; `NO` — в противном случае

> **Важно:** `Trade allowed: YES` НЕ является сигналом на открытие сделки. Это лишь означает, что текущая неопределённость рынка достаточно низкая для принятия решений. Направление сделки трейдер определяет самостоятельно по дополнительным факторам (тренд, уровни поддержки/сопротивления, фундаментальные данные и т.д.).

### 3.5. Алгоритм принятия решения (пример)

1. Посмотрите на **цвет фона**. Если фон серый — **не торгуйте**.
2. Если фон чистый — посмотрите на метку `Trade allowed`. Должно быть `YES`.
3. Если `Signal` > `MinProbabilityToTrade` (0.65) → рынок с большей вероятностью пойдёт **вверх**.
4. Если `1 − Signal` > `MinProbabilityToTrade` (т.е. Signal < 0.35) → рынок с большей вероятностью пойдёт **вниз**.
5. Сопоставьте с вашей торговой стратегией, уровнями, новостным фоном и принимайте решение.

## 4. Параметры и настройки

### 4.1. Основные параметры

| Параметр | По умолчанию | Диапазон | Описание |
|---|---|---|---|
| `LookbackBars` | 200 | 50–1000 | Глубина истории для расчёта априорных вероятностей. Больше = стабильнее, но медленнее адаптируется. |
| `SmoothingPeriod` | 14 | 5–50 | Период SMA-сглаживания отображаемых линий. Меньше = быстрее реагирует, но больше шума. |
| `EntropyThreshold` | 0.65 | 0.1–1.0 | Порог энтропии. Сигнал подавляется, если мгновенная энтропия выше этого значения. Чем ниже порог — тем строже фильтр, меньше ложных сигналов, но больше пропущенных движений. |
| `MinProbabilityToTrade` | 0.65 | 0.51–0.95 | Минимальная пост-байесовская вероятность для подтверждения направления. Выше = сигналы реже, но надёжнее. |
| `UpdateIntervalTicks` | 50 | 10–500 | Интервал (в тиках) полного пересчёта статистики на лету. Меньше = чаще пересчитывается, больше нагрузка на CPU. |

### 4.2. Визуализация

| Параметр | По умолчанию | Описание |
|---|---|---|
| `ShowEntropyZone` | `true` | Включает/выключает серую фоновую закраску зон высокой энтропии. |
| `SignalLineColor` | `clrDodgerBlue` | Цвет линии Signal Probability. |
| `EntropyLineColor` | `clrOrangeRed` | Цвет линии Entropy. |
| `HighEntropyBgColor` | `clrLightGray` | Цвет фона при высокой энтропии. |

### 4.3. Оптимальные настройки (рекомендация автора)

Параметры **по умолчанию** подобраны как сбалансированный вариант для большинства рыночных условий. Они обеспечивают приемлемый компромисс между чувствительностью и фильтрацией шума.

**Если вы хотите адаптировать индикатор под свой стиль:**

- **Скальпинг (M1–M5):** уменьшите `LookbackBars` до 100, `SmoothingPeriod` до 5–7, `EntropyThreshold` до 0.55.
- **Внутридневная торговля (M15–H1):** настройки по умолчанию.
- **Свинг-трейдинг (H4–D1):** увеличьте `LookbackBars` до 300–500, `SmoothingPeriod` до 21–34.

## 5. Рекомендованные валютные пары и таймфреймы

Индикатор **не привязан к конкретному инструменту** — он работает на любых валютных парах, индексах, металлах, криптовалютах и акциях, где доступны цены Open/Close.

### 5.1. Почему подходят все пары?

Математический аппарат CPS (байесовский вывод + энтропия) является **универсальным статистическим методом**. Он анализирует исключительно соотношение бычьих/медвежьих баров и их последовательности — без привязки к волатильности, спреду или ликвидности конкретного инструмента.

### 5.2. Теоретически предпочтительные пары

С точки зрения информационной теории, индикатор наиболее эффективен на инструментах с **выраженной направленностью движения** (трендовых), поскольку на них априорные вероятности будут более устойчивы, а энтропия — ниже:

| Категория | Пары | Причина |
|---|---|---|
| **Major Forex** | EURUSD, GBPUSD, USDJPY, AUDUSD | Высокая ликвидность, меньше шума, хорошо выражены трендовые периоды. |
| **Cross-курсы** | EURGBP, EURJPY, GBPJPY | Хорошие трендовые движения, но могут быть «шумнее» мажоров. |
| **Индексы** | US30 (DJIA), US500 (S&P 500), GER40 (DAX) | Долгосрочные тренды. Рекомендуется H1 и выше. |
| **Золото** | XAUUSD | Сильные трендовые импульсы. Работает с увеличением LookbackBars (300+). |

### 5.3. Рекомендация по самостоятельному тестированию

> ⚠️ Данный раздел содержит **теоретические рекомендации**, а не результаты тестирования. Поскольку у автора на машине разработки (macOS) отсутствует терминал MetaTrader 5, практическое тестирование бэктестом не проводилось.

**Стратегия самостоятельного подбора оптимальных пар:**

1. Откройте 10–15 популярных пар на дневном графике.
2. Накиньте индикатор CPS на каждую с параметрами по умолчанию.
3. В течение 1–2 недель наблюдайте, на каких парах `Trade allowed: YES` появляется чаще, и при этом цена действительно идёт в предсказанную сторону.
4. Ведите дневник наблюдений (или используйте встроенный тестер стратегий MT5 с сигналами CPS как фильтром).
5. Отберите 3–5 пар с наилучшим соотношением «сигнал/шум».

### 5.4. Про тестирование на данном компьютере

Настоящий проект разработан на компьютере под управлением **macOS** без установленного MetaTrader 5. На данной платформе невозможно запустить терминал MT5 и провести бэктестинг или оптимизацию в тестере стратегий. Поэтому следующие аспекты оставлены для самостоятельной проверки пользователем:

- Подбор **оптимальных значений** `LookbackBars` и `EntropyThreshold` под конкретную пару и таймфрейм
- Тестирование в **Strategy Tester** MT5 с использованием сигнала CPS как фильтра
- **Оптимизация** параметров (встроенный оптимизатор MT5)

## 6. Технические детали

### 6.1. Буферы индикатора

| Буфер | Назначение | Отображается |
|---|---|---|
| `SignalBuffer[]` | Сглаженная апостериорная вероятность (plot 0) | Да |
| `EntropyBuffer[]` | Сглаженная энтропия Шеннона (plot 1) | Да |
| `SignalRawBuffer[]` | Сырая (несглаженная) вероятность | Нет (для отладки) |

### 6.2. Ключевые особенности реализации

- **Не перерисовывает историю** — использует только данные баров **старше** текущего. Сигналы не меняются задним числом.
- **Не содержит торговых операций** — индикатор только визуализирует вероятности, не открывает/закрывает сделки. Полностью безопасен для счёта.
- **Защита от деления на ноль** — все знаменатели проверяются перед делением.
- **Динамическое управление объектами** — прямоугольники фона создаются/обновляются/удаляются по мере необходимости, не засоряя график.
- **Оптимизация `OnCalculate`** — полный пересчёт статистики только при новом баре или по истечении `UpdateIntervalTicks`; на обычных тиках обновляется только SMA-сглаживание.
